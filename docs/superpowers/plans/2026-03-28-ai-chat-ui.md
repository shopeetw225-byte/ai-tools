# AI 聊天介面 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把 `apps/web` 的聊天室升級為符合 React 19 + Vite 7 + Tailwind CSS v4 的正式聊天介面，支援 Markdown、串流狀態、多行輸入、錯誤與 retry。

**Architecture:** 先升級 `apps/web` 的工具鏈與測試基礎設施，再以 TDD 重寫 `useChat` 的狀態模型與 SSE 相容邏輯，最後把現有 `ChatPanel` 拆成 `ChatConversation`、`ChatMessageList`、`ChatMessageBubble`、`ChatComposer` 等聚焦元件並接回頁面。`CMPAA-20` 以前端相容目前後端 SSE 為主，不在此 task 內強制改動 `apps/api`；若後端要切到新的 frame 契約，交由 [CMPAA-21](/CMPAA/issues/CMPAA-21) 處理，前端只預留可擴充 parser。

**Tech Stack:** React 19、Vite 7、Tailwind CSS v4、TypeScript、Vitest、React Testing Library、Cloudflare Workers SSE API

- [ ] **Step 3: 建立共用型別與 SSE test helper**

```ts
export type ChatStatus = 'idle' | 'submitting' | 'streaming' | 'error'
export type AssistantMessageState = 'streaming' | 'complete' | 'error' | 'aborted'
```

```ts
export function createSseStream(frames: string[]) {
  return new ReadableStream({
    start(controller) {
      frames.forEach((frame) => controller.enqueue(new TextEncoder().encode(frame)))
      controller.close()
    },
  })
}
```

- [ ] **Step 4: 以最小實作讓 hook 測試通過**

```ts
const activeRequestIdRef = useRef<string | null>(null)

function isActiveRequest(requestId: string) {
  return activeRequestIdRef.current === requestId
}
```

重點：
- `sendMessage` 只送已完成 transcript + 新 user 訊息
- `retryLastMessage` 沿用 `conversationId`
- `clearChat` abort 後立刻 reset
- abort 前沒有收到 chunk 時移除 placeholder
- abort 前已有 partial chunk 時保留內容並標記 `aborted`
- 先相容目前後端 SSE：`{ text, conversationId, error, model }` + `[DONE]`
- parser 內部要留出分支，未來可平滑接 `meta/chunk/error/done`
- 舊 request 的 `chunk/error/finally` 一律先檢查 `requestId`
